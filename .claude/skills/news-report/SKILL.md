# 输出报告

将新闻爬取、真实性验证、时间线梳理、趋势预测等分析结果生成报告。
严格按照下面的目录结构生成文件，一个新闻一个md文件；
每个md文件中严格按照news_xxx.md（单篇文章详情）模块来生成文件。

## 目录结构

```
reports/
├── index.md                    # 主入口，所有分类总览，不包含真实性、时间线、预测
├── {分类}/
│   └── {日期}/
│       ├── index.md           # 分类索引，该日期下所有文章列表
│       ├── {新闻名称}.md        # 文章详情（标题、摘要、内容、图片、真实性、时间线、预测）
│       ├── {新闻名称}.md
│       └── ...
```

## 报告内容
### index.md（查询到的所有新闻的总览）
包含以下模块：
1. **分类总览**
2. 新闻列表
   - 标题
   - 单篇文章详情md文档的链接

### news_xxx.md（单篇文章详情）
包含以下模块：

1. **基础信息**
   - 标题
   - 来源
   - 发布时间
   - 原文链接（原文详情的网页链接URL）
   - 摘要（100-200字）

2. **正文内容**
   - 完整正文（网页解析获取的内容）
   - 原文图片（保留网页解析获取的内容图片）

3. **真实性验证**
   - 来源可信度
   - 交叉性验证，以及验证依据及相关网络链接地址

4. **时间线梳理**
   - 相关事件时间线
   - 事件发展脉络（包含时间线相关的具体新闻网页链接URL）

5. **趋势预测**
   - 短期/中期预测
   - 关键观察指标

### index.md（索引文件）
- 文章标题列表
- 发布时间
- 真实性评分
- 快速跳转链接

## 使用方式

```
/news-report <分类> <日期>
例如：/news-report 科技 20260131
```
